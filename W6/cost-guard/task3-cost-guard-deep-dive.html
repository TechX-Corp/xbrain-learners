Architecture

Operating loop: schedule → scan → decide → stop → prove

Event-driven control
01
EventBridgeRule invokes Lambda on a schedule. Default is daily; can switch to rate(5 minutes) for demo.

LambdaCreates EC2/RDS clients in current region and paginates inventory APIs.
03
Tag contractNormalize AWS tag list into dict and evaluate the protection rule.

Stop actionCalls StopInstances or StopDBInstance only for unprotected running resources.

Lambda internals

Code được tách thành 3 trách nhiệm nhỏ.

Python 3.12

1. Normalize tags

AWS trả tag dạng list object. Mình convert thành dict để rule check đơn giản và testable.

2. Scan service-specific inventory

EC2 tags nằm trong DescribeInstances; RDS phải gọi thêm ListTagsForResource bằng ARN.

3. Stop only unprotected

Chỉ gọi Stop API khi resource đang chạy và fail tag contract.

lambda_function.py
def _is_protected(tags):
    keep_value = tags.get("keep", "").lower()
    env_value = tags.get("Environment", "").lower()
    return keep_value == "true" or env_value == "dev"

if not _is_protected(tags):
    ec2.stop_instances(InstanceIds=[instance_id])

if not _is_protected(tags):
    rds.stop_db_instance(DBInstanceIdentifier=db_id)
Implementation noteRDS tag shape differs from EC2
Least privilege IAM

Role chỉ có quyền nhìn inventory và stop target.

No admin policy

Allowed actions

  • ec2:DescribeInstances
  • ec2:StopInstances
  • rds:DescribeDBInstances
  • rds:ListTagsForResource
  • rds:StopDBInstance
  • logs:CreateLogStream, logs:PutLogEvents

Why Describe uses *

AWS inventory APIs như DescribeInstances không hỗ trợ resource-level ARN restriction, nên cần resources = ["*"].

Stop is scoped by ARN

StopInstances và StopDBInstance được scope về instance/db ARN trong current account + region.

Log group managed by Terraform

Terraform tạo log group trước, nên Lambda không cần logs:CreateLogGroup.

IAM designDescribe/List + Stop only

Q&A drill

Nếu trainer hỏi, trả lời theo 5 ý này.

Presentation ready

1. Lambda quyết định stop thế nào?

Nó normalize tag rồi check keep=true OR Environment=dev. Fail cả hai thì stop.

2. Vì sao cần schedule?

Budget cost data trễ 8–24h; schedule scan resource state trực tiếp nên chặn lãng phí sớm hơn.

3. Least privilege ở đâu?

Role không có admin. Chỉ Describe/List inventory, Stop target, và ghi Lambda logs.

4. Chứng minh stop bằng gì?

Before/after state + CloudTrail StopInstances/StopDBInstance với principal là Lambda role.

5. RDS khác EC2 chỗ nào?

EC2 trả tag trong instance object; RDS cần gọi ListTagsForResource bằng DB ARN.
